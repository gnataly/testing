## Тестирование и отладка ПО (ИУ7 7 семестр)
Над проектом велась работа в рамках курса ППО и курсовой работе по БД, затем в рамках курсов по тестированию и вебу. Версии проекта в зависимости от требований курса различаются.

## Проект театрального центра
Fullstack-разработка web-приложения от идеи и написания ТЗ до полной реализации бэкенд и фронтенд частей и тестирования.
Технологический стек проекта:
1. Frontend (SPA):
Язык: TypeScript.
Фреймворк: React.js.
2. Backend (C#):
Язык: C#.
Фреймворк: ASP.NET Core.
3. СУБД: PostgreSQL.
Проектирование согласно принципам чистой архитектуры.
Целью проекта было разработать web-сервис с собственной базой данных, backend-частью, API и пользовательским интерфейсом.
Работа над проектом включала следующие этапы:
- Разработка идеи, проектирование базы данных и приложения, написание ТЗ (в том числе: описание идеи, описание предметной области, анализ аналогичных решений, обоснование целесообразности и актуальности проекта, описание ролей, создание use-case диаграммы, создание ER-диаграммы, проработка пользовательских сценариев, формализация ключевых бизнес-процессов в нотации BPMN, выбор и обоснование технологического стека, создание UML-диаграммы классов с разбиением на компоненты, создание UML-диаграммы сущностей базы данных и транспортных сущностей (DTO)).
- Программная реализация компонентов бэкенда на C# (Domain, Repositories, Services, Controllers, DTO).
- Контейнеризация в Docker.
- Проектирование REST API и подключение Swagger.
- Тестирование backend, в том числе: написание Unit-тестов интеграционных тестов для компонентов доступа к данным и бизнес-логики проекта на C# с автоматической генерацией отчета при помощи allure; написание E2E-теста для демонстрационного сценария MVP; реализация E2E тестов с полной проверкой аутентификации пользователей и плановой смены пароля; реализация запуска тестов в виде отдельного контейнера docker; интеграция в проект средство трассировки и мониторинга (opentelemetry) для сбора информации, необходимой для отладки проекта; интеграция в проект средства статического анализа для проверки цикломатической сложности и сложности по Холстеду; интеграция проверки code style; настройка CI/CD. Для запуска тестов инициализируется отдельный инстанс хранилища.
- Программная реализация mock-сервера для интеграции с внешним сервисом CloudPayments и реализация E2E-сценария тестирования этой интеграции
- Настройка асинхронного web-сервера для работы web-приложения в части маршрутизации NGINX.
- Разработка макета пользовательского интерфейса в Figma (портреты пользователей в соответствии с use-case диаграммой; доска настроения; дизайн-язык - описание общей стилистики/дизайна, в том числе цветовая палитра, типографика, размеры, примеры геометрии, примеры иконок и иллюстраций; сам макет, в котором проработаны все экраны приложения).
- Программная реализация пользовательского графического интерфейса (frontend) в виде одностраничного приложения (SPA) на TypeScript + React (+ HTML + CSS) с выделением слоев взаимодействия с API, ключевой бизнес-логикой и представления и использованием иерархической системой ui-компонентов (панели, кнопки, поля, формы ввода, списки, экраны) и поддержкой полноценной маршрутизации. Покрытие интеграционными тестами ключевой логики интерфейса с использованием jest.
